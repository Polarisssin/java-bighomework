## v1.6.0 — 安全与业务规则加固

**目标**：对照实训/答辩常见的「权限与安全」检查项，补齐 RBAC、鉴权与易出 Bug 的并发场景。

| 问题 / 风险 | 原因 | 修复 |
|-------------|------|------|
| 未登录可访问部分 API | 云函数未统一校验 Token | 白名单外路由 `requireAuth`；审批/用户 `requireAdmin` |
| Spring 接口权限不清晰 | 仅依赖菜单隐藏 | `@PreAuthorize` + 401/403 处理器 |
| 同一床位被两人同时入住 | 无行锁/事务 | 入住流程 `withTransaction` + `FOR UPDATE` |
| 护理次数扣成负数 | 未校验剩余次数 | 扣减前校验 `remaining`，不足则 400 |
| 审批可重复点击通过 | 未校验状态 | 已审记录拒绝再次操作 |
| 删用户、退住等误操作 | 无二次确认 | `confirm-danger.ts` + 关键按钮确认框 |
| 前端刷新后仍进内页但接口 401 | 仅路由未拦 | `router` 守卫 + `request` 拦截器跳登录 |

**涉及提交**：`d8b8537`（`fix(security): RBAC, business rules, confirmations, and transactions`）

**验证建议**：分别用 `admin`、`nurse01`、`health01` 登录，确认菜单与接口拒绝符合角色；并发测试同一空床两次入住应仅成功一次。

## v1.7.0 — 密码加密与 GitHub 推送脚本

**目标**：满足「密码不可明文落库」；完善本地/云端迁移与仓库同步脚本。

| 项 | 实现 |
|----|------|
| 库存明文密码 | `sql/migrate-password-bcrypt.sql` + `migrate-password-bcrypt.ps1` 批量改为 BCrypt |
| 新用户/重置 | Spring `UserController`、云函数用户接口写入哈希 |
| 旧账号仍可登录 | 校验时识别明文，成功后写回 BCrypt（双端一致） |
| Git push 失败 `Connection reset` | 全局 `git http.proxy=7890` 易导致 TLS 中断；`push-github.ps1` 清除代理并可选临时关系统代理；需要时代理推送：`git -c http.proxy=http://127.0.0.1:7890 push` |
| 实训资料入库 | `基础编程实训/` 答辩 PPT、演示 mp4 等（PDF 仍 `.gitignore`） |

**涉及提交**：`ebb2b79`（BCrypt）、`f00abaf`（脚本与资料）

**验证**：云库迁移后 `admin/admin` 登录；`git push origin main` 或 `.\scripts\push-github.ps1`。

**对照清单更新**：`docs/需求规格对照清单.md` 中「密码安全」已由明文改为 BCrypt + Token（云函数/Spring 均已加固）。
