# 迭代修 Bug 记录（可用于实训报告 / GitHub 展示）

本文档按时间线记录主要问题、原因与修复，与 `CHANGELOG.md` 版本号对应。

---

## v1.0.0 — 基线功能

**目标**：完成需求规格中的客户、床位、护理、管家、审批等主流程。

**技术选型**：

- 前端：Vue 3 + Vite + Element Plus
- 后端：`backend/` Spring Boot；线上 `cloudfunctions/elderCareApi`
- 数据：`sql/init.sql`（18 张表）

---

## v1.1.0 — 部署上云

| 问题 | 处理 |
|------|------|
| 仅本地可访问 | 接入腾讯云 CloudBase 静态托管 + 云函数 |
| 云函数连不上 MySQL | VPC 子网 + 环境变量 `DB_*` |

## v1.2.0 — 中文乱码（典型 Bug）

| 现象 | 原因 | 修复 |
|------|------|------|
| 菜单、标题显示 `????` | 构建链 `charset: ascii` + 文件被错误保存为乱码 | `frontend/src/constants/ui-text.ts` 统一中文；`scripts/ascii-dist.mjs` 构建后转义；修复 `MainLayout.vue` |
| 登录成功页异常符号 | 实验性动效与编码叠加 | 回退登录过渡动画，简化路由切换 |

**验证**：重新 `npm run deploy:hosting`，浏览器 Ctrl+F5 强刷。

## v1.3.0 — 体验与权限

## v1.4.0 — 对照计划书补缺

对照 `01.东软颐养中心-需求规格说明书.pdf`，补齐：

- `beddetails` 床位使用周期
- 护理级别 CRUD、服务关注续费
- 用户新增（默认密码手机号后 6 位）
- 审批人/审批时间
- 入住默认「自理老人」筛选

**遗留说明**：膳食模块、云侧 JWT 加密等见 `docs/需求规格对照清单.md`。
