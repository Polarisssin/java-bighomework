# 东软颐养中心 — 需求规格对照清单

对照文档：`基础编程实训/01.东软颐养中心-需求规格说明书.pdf`

## 已补齐（本次）

| 模块 | 规格要求 | 实现说明 |
|------|----------|----------|
| 入住登记 | 默认筛选自理老人 | `CheckinView` 默认 `elderlyType: self` |
| 床位管理 | beddetails 记录、按条件查询、仅改结束日 | `GET/PUT /api/beds/details`，`ManageView` 记录表 |
| 入住/调换/退住 | 床位使用周期与合同到期联动 | 入住、换床、退住、合同到期更新时写/关 beddetails |
| 护理级别 | 新增级别、启用/停用 | `POST/PUT /api/nurse/level`，`LevelView` |
| 客户护理设置 | 移除级别级联、批量购买级别项目 | `DELETE .../level`、`POST .../level-items-batch` |
| 服务关注 | 到期/欠费/数量正常/未到期、续费、移除 | `GET /api/health/service-items`、`PUT .../renew`，`ServiceView` |
| 用户管理 | 新增用户、默认密码手机号后6位 | `POST /api/users`，`UserManageView` |
| 审批 | 审批人、审批时间 | 外出/退住审核写入 `auditperson`、`audittime`，列表展示 |
| 菜单 | 按角色从库加载 | `menusForRoleFromDb`（menu/rolemenu 有数据时生效） |

## 仍与规格或实训文档有差异（说明）

| 项 | 说明 |
|----|------|
| 技术栈 | 规格示例为 Vue-cli + Vuex；本项目为 Vite + Pinia |
| 线上 API | 生产为云函数 Node；本地可选用 Spring Boot `backend/` |
| 膳食管理 | meal/food/customerpreference 表未建业务页面 |
| 密码安全 | 云环境仍为明文密码 + 简单 token；Spring 侧可用 JWT |
| 3D 身体详情 | 超出原规格，为扩展功能 |

## 部署后建议操作

1. 云数据库若缺 `beddetails`：访问任意床位接口会自动 `CREATE TABLE IF NOT EXISTS`。
2. 若需 DB 菜单：执行 `sql/cloud-init-data.sql` 中 menu/rolemenu 段后重新登录。
3. 重新部署：`cd frontend && npm run deploy:hosting`（含云函数）。

## 符合度（核心业务流程）

- **客户 / 床位 / 护理 / 服务关注 / 用户 / 审批**：约 **90%+** 与规格 §3.2、§5 主流程一致。
- **膳食、报表打印、完整 RBAC 菜单维护界面**：未纳入本次范围。
